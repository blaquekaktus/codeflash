## `blaquekaktus/codeflash` (this repo)

- **Exact purpose.** Public marketing / landing site for the CodeFlash product. Three free-sample pack HTML files + nine pre-purchase bundle HTML previews + landing page + Austrian legal pages. Per `CLAUDE.md`: *"marketing/landing site for the CodeFlash spaced-repetition product (`flashcards-programming-app`). Funnels discovery → product signup."*
- **Tech stack actually used.** Hand-written static HTML/CSS/JS in `index.html` (37 KB single file). No build step. No JS framework. Fonts loaded from Google Fonts (Inter + JetBrains Mono). Dark theme with forest-green accent.
- **Deployment target — verified from CI.** **Unclear.** The repo claims GitHub Pages in `README.md` but there is **no** `pages-build-deployment`, `deploy`, or equivalent workflow under `.github/workflows/`. The only workflow present is `.github/workflows/file-size-guard.yml` (blocks PRs >10 MB). Pages is likely enabled via repo Settings → Pages, but that's not captured in committed CI. The stated public URL `https://codeflash.at` appears in `README.md` only; `index.html` Open Graph URL still points at `https://blaquekaktus.github.io/flashcards-programming-app/` (the OLD GitHub Pages URL for the product repo). This is **drift**.
- **Public vs. private.** Public (visible to anyone on GitHub). Per `CLAUDE.md`: *"the entire repo is the public surface"* (see `.layer0-allow`).
- **Last meaningful commit + activity rhythm.** Last commit `310269d` on 2026-04-19 (hook tightening). Product / content last changed `66a6c10` 2026-04-17 (merge of Python + OWASP packs). Activity: ~15 commits between 2026-04-11 and 2026-04-19, mostly around CLAUDE.md framework rollout, legal pages, hook/allow-list, pack additions. Low-frequency but not abandoned.

## `blaquekaktus/flashcards-programming-app`

- **Exact purpose.** The actual CodeFlash product — Vite/React storefront + Supabase (Postgres + Auth) + Stripe/Lemon Squeezy checkout + Python pack generator (`engine/gen_pack.py`) + pack library (~80 HTML files under `public/packs/**`). Also ships its own marketing `Landing.jsx` + `Bundles`/`BundleDetail`/`TopicDetail`/`Study`/`Dashboard`/`Leaderboard`/`Auth`/`Checkout`/`Success`/`Impressum` pages (`src/pages/*.jsx`).
- **Tech stack actually used.** Vite 5 + React 18 + React Router 6 + `@supabase/supabase-js` (SPA, localStorage auth) + `@stripe/stripe-js` (client-side only) + `simple-icons`. Payment-provider abstraction in `src/lib/payments.js` supports Stripe, Gumroad, Lemon Squeezy via `VITE_PAYMENT_PROVIDER`. Two Supabase Edge Functions: `supabase/functions/{create-checkout,stripe-webhook}/index.ts`. Python 3 stdlib for engine. `package.json.name` renamed from `"codeflash"` to `"flashcards-programming-app"` in PR #12.
- **Deployment target — verified from CI.** Vercel, via `vercel.json` (SPA rewrite `/(.*)` → `/index.html`) + handoff docs (`docs/SESSION_HANDOFF_2026-04-13_deploy.md`). No deploy workflow in `.github/workflows/` — Vercel's GitHub integration auto-deploys `main`. Only CI workflow is the same `file-size-guard.yml` (SHA `3047914d` — identical file across both repos). No test CI, no lint CI, no build CI.

Every SQL file, every Supabase URL, and the committed anon key in `docs/NEXT_SESSION.md` is visible to anyone.
- **Last meaningful commit + activity rhythm.** Last commit `8a296fb` on 2026-04-20 (favicon placeholder).
